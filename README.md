# CodingStyleChecker

Outil TrysCode planifié pour contrôler les conventions de style des projets.

## État

Le dépôt est au stade `planned` : aucun langage, build, test ou déploiement
n'est encore défini. La CI valide uniquement le socle documentaire minimal et
l'absence d'erreurs d'espaces dans les changements Git.

## Responsabilité

Ce dépôt portera le moteur de validation des conventions de code. Il ne doit ni
exécuter de déploiement ni accéder aux secrets des environnements TrysCode.

## Prochaine étape

Définir les langages pris en charge, leurs règles et les tests associés avant
d'activer une CI de compilation et de validation fonctionnelle.
